用纯 CSS 实现故障艺术效果

Glitch 美学不只是视觉噱头，它是一种对"完美"的反叛。
本文探索如何用 clip-path 和 animation 实现真实的故障感。

故障艺术（Glitch Art）起源于数字媒体的错误与失真——那些本该被修复的 bug，却意外地创造出了独特的美感。在 UI 设计中，这种风格被大量运用于标题、Logo 和交互效果。

核心原理

CSS Glitch 效果的核心是 clip-path 属性配合 @keyframes。通过创建两个伪元素，分别赋予不同颜色和偏移，再用 clip-path 切割出不同区域，就能模拟出信号干扰的视觉效果。

.glitch {
  position: relative;
}
.glitch::before, .glitch::after {
  content: attr(data-text);
  position: absolute;
  inset: 0;
}
.glitch::before {
  color: #ff3c6e;
  animation: glitch1 3s infinite;
}
.glitch::after {
  color: #c8ff00;
  animation: glitch2 3s infinite;
}

关键帧动画

动画的关键在于 clip-path: inset() 的随机切割，配合微小的 translate 偏移，制造出"信号跳动"的感觉。

@keyframes glitch1 {
  0%, 100% {
    clip-path: inset(0 0 95% 0);
    transform: translate(-2px, 0);
  }
  20% {
    clip-path: inset(30% 0 50% 0);
    transform: translate(2px, 0);
  }
  60% {
    clip-path: inset(80% 0 5% 0);
    transform: translate(-1px, 0);
  }
}

注意事项

伪元素需要通过 data-text 属性获取文本内容，这意味着你需要在 HTML 中手动设置这个属性。对于动态内容，可以用 JavaScript 自动同步。

性能考量

CSS 动画默认在主线程运行，大量 Glitch 效果可能影响性能。建议只在标题等关键元素上使用，并通过 will-change: transform 提示浏览器开启 GPU 加速。同时，可以用 hover 触发代替持续播放，减少不必要的计算开销。
